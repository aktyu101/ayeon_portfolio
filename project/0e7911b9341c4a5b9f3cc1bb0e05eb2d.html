프로젝트 개요
담당 업무
  • 요구사항 분석 및 기능 명세서 작성
  • UI/UX 설계 및 와이어프레임 제작

이슈
💡
선물하기 이슈: 교환, 환불, 반품 정책이 미비한 상황에서 클라이언트와의 소통을 통해 정책을 정의하고 개발 로직에 반영해야 했으며, 이를 위해 애자일 방법론을 활용하여 신속하고 유연한 대응이 필요했습니다. 개선사항: 선물하기 상품은 일반 상품 주문과 달리 수락, 거절, 대기, 기한만료, 배송 단계와 발송자 및 수령자의 교환, 환불, 반품 각 케이스를 고려해야 하므로, 1차적으로 각 케이스별로 처리 방안을 정리 하였음, 그 후 고객사와의 소통을 통해 최종 정책 수립 후 사전 테스트를 통해 기능 구현하는데 문제점이 없는지 확인하였습니다.
💡
최대혜택가 노출 기능 이슈: 기능 구현 과정에서 특정 결제 금액 이상일 때 사용 가능한 쿠폰이 발급된 경우, 해당 금액을 넘기지 못하면 최대 혜택가와 실 할인 적용 금액이 상이할 수 있는 문제가 확인되었습니다. 이 문제를 해결하기 위해 적절한 처리 방안이 필요했습니다. 개선사항: 쿠폰 등록 단계에 최대 혜택가 포함 여부를 선택할 수 있는 옵션을 라디오 버튼으로 추가하였으며, 디폴트값은 '미포함'으로 설정하였습니다.
배운 점: 기능 구현 과정에서 예상하지 못한 변수가 많았던 경험을 통해, 다양한 상황을 가정한 테스트의 중요성을 깨달았으며, 기획자의 상황 판단력이 작업에 미치는 영향의 크기를 인식하게 되었습니다.

작업내용
- 선물하기 주문 건 부분 환불 불가 처리(이미 주문상품에 대한 정보가 알림톡으로 발송 된 상태이기 때문에 부분 환불 불가) - 선물 주문건의 교환/반품/환불의 경우 선물상태 ‘대기’인 경우 전체 환불만 가능(고객 환불 시 환불접수 상태를 거치지 않고 자동 환불완료 처리 됨) - 환불 신청 페이지 : 수량 조절, 환불 상품 선택 불가 처리(after 가상 요소 추가하여 dim처리 되어 있음)) - 주문상태 ‘결제 실패’ → 결제 중단/실패 건 - 주문상태 ‘대기’ → 결제완료 건 and 환불 처리x and 주문 기한 7일 이내의 건 - 주문상태 ‘수락’ → 수령자 선물 수락(이벤트 시점 : 수령자 정보 작성폼 입력 후 수락 버튼 누른 시점) 건 - 주문상태 ‘거절’ → 수령자 선물 거절(이벤트 시점 : 수령자 정보 작성폼 내 거절 버튼 누른 시점(자동 환불접수 처리)) - 주문상태 ‘기한만료’ → 대기상태의 결제완료 건 and 주문한지 7일이 지난 건 and 환불 처리x * 기한만료, 거절의 경우 환불 상태 확인을 위해 환불접수, 환불완료 상태를 추가로 표시 함 ex) 거절(환불접수) - 주문상태 ‘환불접수’ → 대기 또는 수락 상태에서 환불접수 처리 된 건(고객 환불, 어드민 환불 동일하게 표시) - 주문상태 ‘환불완료’ → 대기 또는 수락 상태에서 환불완료 처리 되 건(고객 환불, 어드민 환불 동일하게 표시) - 선물 주문 상세 : 수령자 배송지 정보 미노출 처리(개인정보 보호)
